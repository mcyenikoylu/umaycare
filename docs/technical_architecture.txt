## 3. Database Architecture

### 3.1 Supabase PostgreSQL

**Versiyon:** PostgreSQL 15  
**Hosting:** Supabase Cloud  
**Features:**
- Row Level Security (RLS)
- Real-time subscriptions (opsiyonel)
- Storage buckets
- Auth integration

**Ana Tablolar:**
```
- profiles                 (Tüm kullanıcılar)
- mother_profiles          (Anne özel verileri)
- doctor_profiles          (Hekim özel verileri)
- babies                   (Bebek kartları)
- appointments             (Randevular)
- availability_slots       (Hekim müsaitlik)
- availability_exceptions  (Müsaitlik istisnaları)
- treatment_records        (Tedavi kayıtları)
- prescriptions            (Reçeteler)
- lab_tests                (Laboratuvar testleri)
- articles                 (Makale içerik)
- videos                   (Video içerik)
- subscriptions            (Abonelikler)
- subscription_packages    (Paketler)
- chat_sessions            (AI chat oturumları)
- chat_messages            (Chat mesajları)
- healthcare_locations     (Hastane/Eczane)
- doctor_assistants        (Doktor-Asistan ilişkisi)
- mother_partners          (Anne-Eş ilişkisi)
- partner_baby_access      (Eş'in bebek erişimi)
```

**İndeksler:**
```sql
-- Performans için kritik indeksler
CREATE INDEX idx_profiles_role ON profiles(role);
CREATE INDEX idx_babies_mother_id ON babies(mother_id);
CREATE INDEX idx_appointments_doctor_date ON appointments(doctor_id, appointment_date);
CREATE INDEX idx_appointments_status ON appointments(status);
CREATE INDEX idx_subscriptions_user ON subscriptions(user_id);
```

**API Deploy Workflow:**
```yaml
name: Deploy API

on:
  push:
    branches: [main]
    paths:
      - 'apps/api/**'
      - 'packages/**'

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Node
        uses: actions/setup-node@v3
        with:
          node-version: '18'
          cache: 'npm'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Build
        run: npm run build --workspace=apps/api
      
      - name: Deploy to Hetzner
        uses: appleboy/scp-action@master
        with:
          host: ${{ secrets.HETZNER_HOST }}
          username: ${{ secrets.HETZNER_USER }}
          key: ${{ secrets.HETZNER_SSH_KEY }}
          source: "apps/api/.next,apps/api/package.json"
          target: "/var/www/umaycare-api"
      
      - name: Restart PM2
        uses: appleboy/ssh-action@master
        with:
          host: ${{ secrets.HETZNER_HOST }}
          username: ${{ secrets.HETZNER_USER }}
          key: ${{ secrets.HETZNER_SSH_KEY }}
          script: |
            cd /var/www/umaycare-api
            npm install --production
            pm2 restart umaycare-api
```

## 8. Security Architecture

### 8.1 Authentication Flow

```
1. User registers
   ├─> Supabase creates auth.users entry
   ├─> Email confirmation sent (MailJet)
   └─> Profile created in profiles table

2. User confirms email
   └─> Account activated

3. User logs in
   ├─> Supabase validates credentials
   ├─> JWT token issued
   └─> Refresh token stored

4. API requests
   ├─> Bearer token in Authorization header
   ├─> Middleware validates JWT
   ├─> User role checked
   └─> Request processed
```

### 8.2 Row Level Security (RLS)

**Example Policies:**
```sql
-- Mothers can only see their own babies
CREATE POLICY "mothers_own_babies" ON babies
FOR SELECT USING (
  mother_id IN (
    SELECT id FROM mother_profiles WHERE id = auth.uid()
  )
);

-- Partners can see authorized babies
CREATE POLICY "partners_authorized_babies" ON babies
FOR SELECT USING (
  id IN (
    SELECT baby_id FROM partner_baby_access 
    WHERE partner_id = auth.uid()
  )
);

-- Doctors can see their patients' babies
CREATE POLICY "doctors_patient_babies" ON babies
FOR SELECT USING (
  id IN (
    SELECT baby_id FROM appointments 
    WHERE doctor_id = auth.uid()
  )
);
```

### 8.3 API Security

**Rate Limiting:**
```typescript
// middleware/rate-limit.ts
import { Ratelimit } from '@upstash/ratelimit';
import { Redis } from '@upstash/redis';

const ratelimit = new Ratelimit({
  redis: Redis.fromEnv(),
  limiter: Ratelimit.slidingWindow(10, '10 s'),
});

export async function rateLimitMiddleware(request: NextRequest) {
  const ip = request.ip ?? 'anonymous';
  const { success } = await ratelimit.limit(ip);
  
  if (!success) {
    return new Response('Too many requests', { status: 429 });
  }
}
```

**Input Validation:**
```typescript
// Example with Zod
import { z } from 'zod';

const babySchema = z.object({
  full_name: z.string().min(2).max(100),
  date_of_birth: z.string().datetime(),

  baby_type: z.enum(['single', 'twin', 'triplet', 'quadruplet'])
});

export async function POST(request: NextRequest) {
  const body = await request.json();
```

## 9. Performance Optimization

### 9.1 Database Optimization

- **Indexes** on frequently queried columns
- **Connection pooling** (Supabase handles this)
- **Query optimization** (use EXPLAIN ANALYZE)
- **Pagination** for large datasets

### 9.2 API Optimization

- **Response caching** (Next.js cache)
- **API route optimization**
- **Lazy loading** of related data
- **GraphQL** consideration for future
